jQuery - El método noConflict()

¿Qué pasaría si quisiéramos utilizar otros frameworks en nuestra página mientras seguimos utilizando jQuery?

jQuery y otros FrameWorks JavaScript

Como ya sabemos jQuery utiliza el signo $ como atajo jQuery.

Hay muchos otros frameworks JavaScript populares como: Angular, Backbone, Ember, Knockout, etc.

¿Qué pasa si otro framework JavaScript también utiliza el signo $ como atajo?

Si dos frameworks diferentes están usando el mismo atajo, uno de los dos podría dejar de funcionar.

El equipo jQuery ya pensó en ello e implementó el método noConflict().

El método jQuery noConflict()

El método noConflict() suelta la retención del identificador de atajo $, así otros scripts pueden utilizarlo.

Podemos, por supuesto, seguir usando jQuery, simplemente escribiendo el nombre completo (jQuery) en lugar del $:

Ejemplo


  $.noConflict();
  jQuery(document).ready(function(){
    jQuery("button").click(function(){
      jQuery("p").text("jQuery is still working!");
    });
  });

También podemos crear nuestro propio atajo ("shortcut") fácilmente. El método noConflict() devuelve una referencia a jQuery que podemos guardar en una variable para uso posterior. Aquí va un ejemplo:

Ejemplo


  var jq = $.noConflict();
  jq(document).ready(function(){
    jq("button").click(function(){
      jq("p").text("jQuery is still working!");
    });
  });

Si tenemos un bloque de código jQuery que utiliza el $ y no queremos cambiar todo, podemos pasar el signo $ como un parámetro en el método ready. Esto no permite acceder a jQuery usando $ dentro de esta función, fuera de ella tendremos que seguir utilizando "jQuery".

Ejemplo


  $.noConflict();
  jQuery(document).ready(function($){
    $("button").click(function(){
      $("p").text("jQuery is still working!");
    });
  });
