jQuery - Los métodos AJAX get() y post()

Los métodos jQuery get() y post() son utilizados para solicitar datos al servidor con una solicitud HTTP GET o POST.

Solicitud HTTP: GET vs. POST

Dos métodos comunmente usados para una solicitud-respuesta entre un cliente y un servidor son: GET y POST.

  • GET - Solicita datos a un recurso específico
  • POST - Envía datos a ser procesados a un recurso específico

    • GET es usado básicamente para solamente obtener datos de un servidor. Nota: el método GET puede devolver datos almacenados en caché.

    POST puede también ser utilizado para obtener datos de un servidor. Sin embargo, el método POST NUNCA almacena datos en caché y es usualmente utilizado para enviar datos junto a la solicitud.

    El método jQuery $.get()

    El método $.get() solicita datos de un servidor con una solicitud HTTP GET.

    Sintaxis:

    
  $.get(URL,callback);

    El parámetro obligatorio "URL" especifica la URL que queremos solicitar.

    El parámetro opcional "callback" es el nombre de una función a ser ejecutada y la solicitud tiene éxito.

    El ejemplo siguiente usa el método $.get() para obtener datos de un archivo en el servidor:

    Ejemplo

    
  $("button").click(function(){
    $.get("demo_test.asp", function(data, status){
      alert("Data: " + data + "\nStatus: " + status);
    });
  });

    El primer parámetro de $.get() es la URL que queremos solicitar ("demo_test.asp").

    El segundo parámetro es una función callback. El primer parámetro de la función callback mantiene el contenido de la página solicitada y el segundo parámetro mantiene el estado de la solicitud.

    Tip: A continuación se muestra cómo se ve el archivo ASP ("demo_test.asp"):

    
  <%
  response.write("This is some text from an external ASP file.")
  %>

    El método jQuery $.post()

    El método $.post() solicita datos de un servidor utilizand una solicitud HTTP POST.

    Sintaxis:

    
  $.post(URL,data,callback);

    El parámetro obligatorio "URL" especifica la URL que queremos solicitar.

    El parámetro opcional "data" especifica datos a enviar junto a la solicitud.

    El parámetro opcional "callback" es el nombre de una función a ser ejecutada si la solicitud resulta exitosa.

    El ejemplo siguiente utiliza el método $.post() para enviar algunos datos junto a la solicitud:

    Ejemplo

    
  $("button").click(function(){
    $.post("demo_test_post.asp",
    {
      name: "Donald Duck",
      city: "Duckburg"
    },
    function(data, status){
      alert("Data: " + data + "\nStatus: " + status);
    });
  });

    El primer parámetro de $.post() es la URL que queremos solicitar ("demo_test_post.asp").

    Luego, pasamos algunos datos a ser enviados junto a la solicitud (name y city).

    El script ASP e "demo_test_post.asp" lee los parámetros, los procesa y devuelve un resultado.

    El tercer parámetro es una función callback. El primer parámetro esta función mantiene el contenido de la página solicitada, y el segundo mantiene el estado de la solicitud.

    Tip: Así es como luce el archivo ASP ("demo_test_post.asp"):

    
  <%
  dim fname,city
  fname=Request.Form("name")
  city=Request.Form("city")
  Response.Write("Dear " & fname & ". ")
  Response.Write("Hope you live well in " & city & ".")
  %>
