jQuery Traversing - Siblings

Con jQuery podemos recorrer hacia los lados el árbolo DOM para encontrar "hermanos" de un elemento.

Hermanos (Siblings) comparten el mismo padre.

Recorrer hacia los lados el árbol DOM

Hay muchos métodos jQuery útiles para recorrer hacia los lados el árbol DOM:

  • siblings()
  • next()
  • nextAll()
  • nextUntil()
  • prev()
  • prevAll()
  • prevUntil()

    • El método jQuery siblings()

    El método siblings() devuelve todos los elementos "hermanos" del elemento seleccionado.

    El siguiente ejemplo devuelve todos los "hermanos" del <h2>:

    Ejemplo

    
  $(document).ready(function(){
    $("h2").siblings();
  });

    También podemos utilizar un parámetro opcional para filtrar la búsqueda de "hermanos".

    El siguiente ejemplo devuelve todos los elementos "hermanos" de <h2> que son elementos <p>:

    Ejemplo

    
  $(document).ready(function(){
    $("h2").siblings("p");
  });

    El método jQuery next()

    El método next() devuelve el elemento hermano siguiente al elemento seleccionado.

    El siguiente ejemplo devuelve el siguiente hermano de <h2>:

    Ejemplo

    
  $(document).ready(function(){
    $("h2").next();
  });

    El método jQuery nextAll()

    El método nextAll() devuelve todos los elementos hermanos siguientes al elemento seleccionado.

    El siguiente ejemplo devuelve todos los elementos hermanos siguientes de <h2>:

    Ejemplo

    
  $(document).ready(function(){
    $("h2").nextAll();
  });

    El método jQuery nextUntil()

    El método nextUntil() devuelve todos los elementos hermanos siguientes entre dos argumentos dados.

    El siguiente ejemplo devuelve todos los elementos hermanos entre un <h2> y un elemento <h6>:

    Ejemplo

    
  $(document).ready(function(){
    $("h2").nextUntil("h6");
  });

    Los métodos jQuery prev(), prevAll() y prevUntil()

    Los métodos prev(), prevAll() y prevUntil() funcionan de la misma manera que los métodos antes mencionados pero con funcionalidad opuesta: devuelven los elementos hermanos anteriores (recorre los elementos hermanos hacia atrás en el árbol DOM, en vez de recorrerlo hacia adelante).
